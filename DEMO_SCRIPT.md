# 90-second demo

## Setup

- Cache the selected GGUF before recording.
- Start Blackout Buddy with `BLACKOUT_ALLOW_MODEL_DOWNLOAD=0`.
- Open the Quick Help tab and show the green offline-ready chip.
- Disconnect Wi-Fi or unplug Ethernet before the first interaction.

## Script

**0:00-0:12 - Stakes**

"When the grid and cell network fail, cloud assistants fail with them. Blackout
Buddy is a source-grounded emergency guide that runs on the device in front of us."

**0:12-0:28 - Offline proof**

Show the disconnected network indicator. Click **Severe bleeding**. Point out that
the reviewed card appears immediately with numbered actions, warnings, and source.

**0:28-0:48 - Local model**

Type: `An adult is choking and cannot speak or cough.` Click **Get Verified Help**.
Show the source card first, followed by the locally streamed grounded explanation.

**0:48-1:05 - Guided triage**

Open Guided Triage. Give short answers through the four questions. Show the final
action plan and explain that the trace is stored locally with phone/email scrubbing.

**1:05-1:18 - Resilience**

Restart without a model path or briefly show Safe Field-Guide mode. Explain that a
model failure never produces a blank screen.

**1:18-1:30 - Build story**

"Inference is llama.cpp and fully offline. The emergency LoRA is trained as a
one-off Modal GPU job and published to Hugging Face. Training is cloud-assisted;
the emergency product is not."

## Recording checklist

- Keep source attribution visible.
- Do not claim medical validation.
- Show the physical network disconnect.
- Link the Space, model adapter, trace dataset, field notes, and source repository.
